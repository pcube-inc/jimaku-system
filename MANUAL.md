## 1. システム概要

字幕システムは、スタッフの業務管理をLINEアプリ上で行うためのWebアプリです。

### 主な機能

| 機能 | 対象 | 内容 |
|------|------|------|
| シフト管理 | スタッフ・管理者 | シフト希望の提出、確定シフトの閲覧・入力 |
| 起床確認 | スタッフ | 出勤日の朝に起床を報告 |
| 出退勤連絡 | スタッフ | 出勤・退勤の打刻と管理者への自動メール通知 |
| トラブル対応 | スタッフ | 緊急時の第一報・第二報を即時送信 |
| 管理者設定 | 管理者 | スタッフ管理・各種通知設定 |

### 動作環境

- **推奨**: LINEアプリ内ブラウザ（LIFF）
- **その他**: スマートフォン・PCブラウザでも動作（一部機能はLINEログイン不要）

## 3. スタッフ向け機能

### 3.1 シフト希望の提出

**アクセス**: メニュー → 📅 シフト管理 → 「シフト希望」タブ

#### 操作手順

1. スタッフ名を選択（LINEログイン時は自動選択）
2. 画面上部のボタンで希望区分を選択
   - **○ OK** — 出勤可能
   - **△ 応相談** — 条件次第で出勤可能
   - **✕ 休み** — 休み希望
3. カレンダーの日付をタップして希望を入力
   - 灰色のストライプ = シフト対象外曜日（選択不可）
   - 薄く表示 = 過去日（変更不可）
   - 同じ日をもう一度タップ → 解除
4. 画面下部のプログレスバーで入力進捗を確認
5. 「送信する」ボタンをタップ

> **ヒント**: 月の移動は ‹ › ボタンで可能。最大3ヶ月先まで提出できます。

---

### 3.2 起床確認

**アクセス**: メニュー → 🌅 起床確認

出勤日の朝、設定された期限時刻までにボタンを押してください。

#### 操作手順

1. 「起床確認」ページを開く
2. 本日のシフト状況が表示される
   - シフトあり → 「起床確認ボタン」が表示される
   - シフトなし → 確認不要のメッセージが表示される
3. 「起床確認」ボタンをタップ
4. 押下済みの場合は確認時刻が表示される

> **注意**: 期限時刻を過ぎても未確認の場合、管理者にメール通知が送られます。

---

### 3.3 出退勤連絡

**アクセス**: メニュー → 🕐 出退勤連絡

#### 出勤時

1. 「出退勤連絡」ページを開く
2. スタッフ名を確認
3. 「出勤」ボタンをタップ
4. 管理者へ出勤通知メールが自動送信される

#### 退勤時

1. 「出退勤連絡」ページを開く
2. 「退勤」ボタンをタップ
3. 管理者へ退勤通知メールが自動送信される

> 打刻内容はGoogleスプレッドシート「出退勤記録」シートに自動保存されます。

---

### 3.4 トラブル対応

**アクセス**: メニュー → 🚨 トラブル対応

緊急時に2段階で管理者へ報告します。

#### 第一報

1. 「トラブル対応」ページを開く
2. スタッフ名を確認
3. 「第一報を送信」ボタンをタップ
4. 管理者・CCに緊急メールが即時送信される

#### 第二報

1. 第一報送信後、詳細入力欄が表示される
2. トラブルの詳細を入力
3. 「第二報を送信」ボタンをタップ
4. 詳細内容が管理者にメール送信される

> 報告内容はGoogleスプレッドシート「トラブル記録」シートに保存されます。

## 4. 管理者向け機能

### 4.1 管理者ログイン

1. `https://pcube-inc.github.io/jimaku-system/` をブラウザで開く
2. メニュー画面の一番下 **⚙️ 管理者設定** をクリック
3. パスワードを入力してログイン
4. 初期パスワード: `admin1234`（初回ログイン後は必ず変更してください）
5. ログイン状態はブラウザセッション中は保持される

---

### 4.2 スタッフ管理

**アクセス**: 管理者設定 → 「スタッフ」タブ

#### スタッフの追加

1. 画面下部の入力欄に「登録名」を入力（必須）
2. 必要に応じてメールアドレスを入力（任意）
3. 「追加」ボタンをタップ

#### スタッフ情報の編集

各スタッフ行の各フィールドを直接編集できます：

| フィールド | 説明 |
|-----------|------|
| 登録名 | アプリ内で表示される名前（管理者が設定） |
| LINE名 | LINEプロフィールから自動取得（参照用） |
| メール | 連絡用メールアドレス |
| 有効 | チェックを外すとシフト一覧から非表示になる |
| LINE ID | LINE連携済みの場合に自動表示 |

#### スタッフの削除

各スタッフ行の「削除」ボタンをタップ → 確認ダイアログで削除

> **スタッフとLINEアカウントの連携**: スタッフがLINEからアプリを初めて開いた際、登録名とLINE表示名が一致する場合は自動連携されます。一致しない場合は管理者が手動で確認・設定が必要です。

---

### 4.3 シフト希望の確認・確定

#### 希望一覧の確認

**アクセス**: シフト管理 → 「希望一覧」タブ

1. 年月を選択
2. フィルターで表示を絞り込み
   - **全件**: すべての日付
   - **充足**: OK・応相談が2名以上
   - **人手不足**: OK・応相談が1名以下
   - **要注意**: 応相談がいる日
3. 各日付カードで希望スタッフを確認

#### 確定シフトの入力

**アクセス**: シフト管理 → 「確定入力」タブ

1. 年月を選択
2. 各日付行でスタッフと業務種別（業務A / 業務B）を選択
3. 「＋ 追加」で同日に複数スタッフを割り当て可能
4. 「確定シフトを保存」ボタンで一括保存

#### 日付別希望の確認（サブビュー）

「確定入力」タブ内の「日付を選んで希望スタッフを確認」エリアで特定日の希望状況を確認できます。

---

### 4.4 LINEグループへの通知送信

**アクセス**: シフト管理 → 「確定入力」タブ → 下部「LINEグループへシフト通知を送信」

1. 送信したい日付を選択
2. 「今すぐ送信」ボタンをタップ
3. 確定シフトの内容がLINEグループに自動送信される

送信メッセージ例：
```
【シフト連絡】
2026-06-07
本日の担当は
田中さん（業務A）
鈴木さん（業務B）
です。よろしくお願いします。
```

> **前提**: 管理者設定のLINE設定でChannel Access TokenとGroup IDを設定している必要があります。

---

### 4.5 基本設定

**アクセス**: 管理者設定 → 「基本設定」タブ

| 設定項目 | 説明 | 例 |
|---------|------|-----|
| 管理者メールアドレス | 各種通知の送信先 | admin@gmail.com |
| CCメールアドレス | 複数宛先（1行1アドレス） | cc1@gmail.com |
| 起床確認の期限時刻 | この時刻までに確認しないとアラート | 08:00 |
| LINE定型文の送信時刻 | 自動送信時刻（GASトリガー用） | 07:00 |
| シフト対象曜日 | シフト提出・確定の対象曜日 | 月〜金 |
| 管理者パスワード | 変更する場合は新旧両方入力 | — |

---

### 4.6 LINE設定

**アクセス**: 管理者設定 → 「LINE設定」タブ

| 設定項目 | 説明 | 取得場所 |
|---------|------|---------|
| Channel Access Token | LINE Messaging APIのトークン | LINE Developers → チャネル → Messaging API |
| Group ID | 送信先LINEグループのID | LINEグループのWebhookログ等から取得 |

## 5. 初期セットアップ（管理者）

新規環境でシステムを利用開始する手順です。

### ステップ1: Google Apps Scriptのデプロイ

1. `gas/` フォルダ内の各 `.gs` ファイルを、それぞれ個別のGASプロジェクトとして作成
   - `shift.gs` → jimaku-shift
   - `wakeup.gs` → jimaku-wakeup
   - `attendance.gs` → jimaku-attendance
   - `trouble.gs` → jimaku-trouble
   - `setup.gs` → jimaku-setup（admin）
2. 各プロジェクトをWebアプリとしてデプロイ
   - 「デプロイ」→「新しいデプロイ」→「Webアプリ」
   - アクセス権限: **全員**
3. 発行されたURLを `js/config.js` の `GAS_URL` に設定

### ステップ2: スプレッドシートの初期化

1. `gas/setup.gs` プロジェクトのエディタを開く
2. `setupSheets()` 関数を実行
3. 以下のシートが自動作成される：
   - スタッフ一覧
   - シフト希望
   - 確定シフト
   - 起床確認
   - 出退勤記録
   - トラブル記録
   - 管理者設定

### ステップ3: LIFF IDの設定

1. LINE Developersで4つのLIFFアプリを作成
   - shift / wakeup / attendance / trouble 各ページ用
2. 発行されたLIFF IDを `js/config.js` の `LIFF_ID` に設定

### ステップ4: 起床確認の自動チェック設定（任意）

1. `gas/wakeup.gs` プロジェクトのGASエディタを開く
2. 「トリガー」→「トリガーを追加」
3. 関数: `checkUnconfirmed`、イベント: 時間主導型、設定した期限時刻の直後に実行

### ステップ5: 毎朝シフト通知の自動送信設定（任意）

1. `gas/shift.gs` プロジェクトのGASエディタを開く
2. 「トリガー」→「トリガーを追加」
3. 関数: `sendDailyMessage`、イベント: 時間主導型（毎朝設定時刻）

### ステップ6: スタッフ登録

1. 管理者設定 → スタッフタブ → スタッフを追加
2. スタッフが初めてLINEからアプリを開いた際にLINE IDが自動連携される

## 6. システム構成・技術情報

### アーキテクチャ

```
LINE LIFF (フロントエンド)
    │
    ├── index.html           メニュー画面
    ├── shift/index.html     シフト管理
    ├── wakeup/index.html    起床確認
    ├── attendance/index.html 出退勤連絡
    ├── trouble/index.html   トラブル対応
    └── admin/index.html     管理者設定
         │
         ↓ JSONP / fetch (no-cors)
    Google Apps Script (バックエンド)
         │
         ↓
    Google スプレッドシート (データベース)
```

### スタッフ一覧シートの構造（v2スキーマ）

| 列 | フィールド | 内容 |
|----|-----------|------|
| A | line_user_id | LINE ユーザーID（自動登録） |
| B | LINE表示名 | LINEプロフィール名（自動取得） |
| C | 登録名 | 管理者が設定するアプリ表示名 |
| D | メールアドレス | 管理者が登録する連絡先 |
| E | 有効フラグ | TRUE/FALSE |

> アプリ内での表示名は「登録名」優先。登録名が未設定の場合はLINE表示名を使用。

### GASプロジェクト一覧

| プロジェクト | ファイル | 主な機能 |
|------------|---------|---------|
| jimaku-shift | gas/shift.gs | シフト希望・確定シフト・スタッフ管理・LINE送信 |
| jimaku-wakeup | gas/wakeup.gs | 起床確認・未確認チェック（時間トリガー） |
| jimaku-attendance | gas/attendance.gs | 出退勤打刻・メール通知 |
| jimaku-trouble | gas/trouble.gs | トラブル第一報・第二報・メール通知 |
| jimaku-setup（admin） | gas/setup.gs | 管理者設定・スタッフ管理・シート初期化 |

## 7. よくある質問 / トラブルシューティング

### Q. スタッフ名が自動で選択されない

**A.** LINEアカウントとスタッフ情報の連携が完了していない可能性があります。
- 管理者設定 → スタッフタブで当該スタッフの「LINE ID」欄を確認
- 「未連携」の場合: スタッフが一度LINEからアプリを開くと自動連携される
- 連携済みだが名前が出ない: 「有効」チェックがオフになっていないか確認

### Q. シフト送信後に反映されない

**A.** GASの処理に時間がかかる場合があります（通常3〜5秒）。
- 「送信しました」のトースト表示を確認してから少し待ってページを更新
- エラーメッセージが出る場合はネットワーク接続を確認

### Q. 起床確認ボタンが表示されない

**A.** 以下を確認してください：
- 本日の確定シフトに自分の名前が登録されているか（管理者に確認）
- すでに押下済みの場合は「確認済み」と表示される

### Q. 管理者へのメールが届かない

**A.** 以下を確認してください：
- 管理者設定の「管理者メールアドレス」が正しく設定されているか
- GASのMailApp送信上限（無料Gmailは1日100通）に達していないか
- 迷惑メールフォルダに振り分けられていないか

### Q. LINEグループへの送信が失敗する

**A.** 以下を確認してください：
- 管理者設定の「LINE設定」でChannel Access TokenとGroup IDが設定されているか
- Channel Access Tokenの有効期限が切れていないか（LINE Developersで再発行）
- LINE BotがグループにメンバーとしてINVITEされているか

### Q. 管理者パスワードを忘れた

**A.** Googleスプレッドシートの「管理者設定」シートを直接開き、`admin_password` の値を確認・変更してください。
