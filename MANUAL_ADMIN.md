# 管理者マニュアル — P-CUBE 業務連絡システム

## システム概要

字幕スタッフの**シフト管理・起床確認・出退勤連絡・トラブル報告**をLINEとWebで一元管理するシステムです。

| 機能 | スタッフが使う | 管理者が確認する場所 |
|------|--------------|-------------------|
| シフト管理 | LINEまたはブラウザでシフト希望提出 | スプレッドシート「シフト希望」「シフト確定」 |
| 起床確認 | 出勤日の朝にLINEから「起床」ボタン | 未確認はメールで自動通知される |
| 出退勤連絡 | LINEから出勤・退勤ボタン | スプレッドシート「出退勤記録」＋メール通知 |
| トラブル報告 | LINEから第一報・第二報を送信 | スプレッドシート「トラブル記録」＋メール通知 |

## 初期設定（最初に必ずやること）

### 1. 管理者設定ページを開く

ブラウザで以下のURLにアクセスします：

```
https://pcube-inc.github.io/jimaku-system/
```

トップページ下部の **「管理者設定」** カードをクリック → パスワードを入力（初期値: `admin1234`）

### 2. 基本設定を入力する

管理者設定ページで以下の項目を設定してください。

| 設定項目 | 説明 | 例 |
|---------|------|---|
| 管理者メールアドレス | 通知メールの宛先 | admin@example.com |
| CCメールアドレス | 通知メールのCC（複数は`,`区切り） | cc1@example.com,cc2@example.com |
| 起床リマインダー時刻 | LINEでリマインド送信する時刻 | 07:00 |
| 起床確認期限時刻 | この時刻までに未確認なら通知 | 08:00 |
| LINEチャネルトークン | LINE Developersで取得したトークン | （長い文字列） |
| 管理者パスワード | 管理者ページのパスワード | admin1234 |
| シフト対象曜日 | シフト対象の曜日番号（月=1,火=2...） | 1,2,4,5 |

設定後は **「保存」** ボタンをクリックしてください。

### 3. 起床確認トリガーを初期化する

Google Apps Script の wakeup プロジェクトを開き、**`setupTriggers`** を1回だけ手動実行してください。

> 以降は毎日自動で実行されます。手動実行は初回のみです。

## スタッフ管理

### スタッフを追加する

1. 管理者設定ページ → 「スタッフ管理」タブ
2. 「スタッフを追加」ボタンをクリック
3. 登録名・メールアドレスを入力して「追加」

> スタッフがLINEで初回ログインすると、LINE表示名とLINE IDが自動で紐づけされます。

### スタッフを無効化する

スタッフ一覧の「有効」トグルをOFFにすると、そのスタッフはシステムに表示されなくなります（データは保持されます）。

### LINE IDの紐づけ

スタッフがLINEでシステムを開くと、LINE IDが自動登録されます。管理者が「登録名」を設定することで名前が紐づきます。

## シフト管理

### シフト確定の流れ

1. スタッフがシフト希望をLINEから提出 → 「シフト希望」シートに記録
2. 管理者がスプレッドシートの「シフト希望」を確認してシフトを組む
3. 管理者設定ページ → シフト管理 → 「シフトを確定」する
4. 確定されたシフトが「シフト確定」シートに記録される
5. LINEで確定通知がスタッフ全員に送信される（Broadcast）

### カレンダーシートの作成

管理者設定ページのシフト管理から「カレンダーシートを作成」すると、月ごとのカレンダー形式シートがスプレッドシートに追加されます。

## 起床確認の仕組み

```
毎日 07:00（設定可）
  ↓ sendWakeupReminder()
当日シフトのスタッフ全員にLINEプッシュ通知

毎日 08:00（設定可）
  ↓ checkUnconfirmed()
未確認スタッフがいれば管理者メール（＋CC）に通知
```

- リマインダーメッセージとメール本文は管理者設定の「テンプレート」で変更できます
- `{name}` `{deadline}` という変数が使えます

---

## メール通知の種類

| イベント | 件名 | 送信タイミング |
|---------|------|-------------|
| 起床未確認 | 【起床未確認】〇〇さんが未確認です | 毎朝 期限時刻に自動 |
| 出勤 | 【出勤】〇〇さんが出勤しました | スタッフが出勤ボタン押下時 |
| 退勤 | 【退勤】〇〇さんが退勤しました | スタッフが退勤ボタン押下時 |
| トラブル第一報 | 【トラブル第一報】〇〇さんからの報告 | スタッフがトラブルボタン押下時 |
| トラブル第二報 | 【トラブル第二報】〇〇さんからの報告 | スタッフが詳細送信時 |

## よくあるトラブル

### 起床確認メールが届かない
- 管理者設定の `admin_email` が正しいか確認
- wakeup GASの `setupTriggers()` を再実行する

### LINEリマインダーが届かない
- 管理者設定の `line_channel_token` が正しいか確認
- スタッフのLINE IDが「スタッフ一覧」に登録されているか確認

### スタッフがLINEでログインできない
- LINE DevelopersでLIFFが公開状態になっているか確認
- LIFFのエンドポイントURLが正しいか確認

### シフト確定通知が届かない
- LINEチャネルトークンを確認
- Broadcast APIが有効か（Messaging API設定を確認）
