## 2. 初期セットアップ（初回のみ）

> 既にセットアップ済みの場合はこの章をスキップしてください。

### 2-1. Googleスプレッドシートの初期化

1. [GAS管理プロジェクト（jimaku-setup）](https://script.google.com) を開く
2. `setupSheets()` 関数を選択して「実行」
3. 以下のシートが自動作成される：
   - スタッフ一覧 / シフト希望 / 確定シフト / 起床確認 / 出退勤記録 / トラブル記録 / 管理者設定

### 2-2. GASの再デプロイ（コード変更時）

各GASプロジェクトで以下の手順を実行：
1. GASエディタを開く
2. 「デプロイ」→「デプロイを管理」
3. 鉛筆アイコン（編集）をクリック
4. バージョン：「新バージョン」を選択
5. 「デプロイ」ボタンをクリック

### 2-3. GASトリガーの設定（自動通知機能）

**起床未確認アラート（wakeup.gs）：**
1. jimaku-wakeupプロジェクトのGASエディタを開く
2. 左メニュー「トリガー」→「トリガーを追加」
3. 関数：`checkUnconfirmed` / イベント：時間主導型 / 起床確認期限の直後に設定

**毎朝シフト通知（shift.gs）：**
1. jimaku-shiftプロジェクトのGASエディタを開く
2. 「トリガー」→「トリガーを追加」
3. 関数：`sendDailyMessage` / イベント：時間主導型 / 毎朝の通知希望時刻に設定

## 3. 管理者設定へのログイン方法

1. `https://pcube-inc.github.io/jimaku-system/` をブラウザで開く
2. メニュー画面の一番下 **⚙️ 管理者設定** をクリック
3. パスワードを入力してログイン

- **初期パスワード**: `admin1234`
- **初回ログイン後は必ずパスワードを変更してください**（「基本設定」タブ内）
- ログイン状態はブラウザセッション中は保持される

## 4. スタッフの登録・管理

**アクセス**: 管理者設定 → 「スタッフ」タブ

### 新しいスタッフを追加する

1. 画面下部の入力欄に「登録名」を入力（必須）  
   ※ 登録名 = アプリ内で表示される名前（例：田中、鈴木）
2. メールアドレスを入力（任意）
3. 「追加」ボタンをタップ

### スタッフがアプリを初めて開いた後の作業

アルバイトが初めてLINEからアプリを開くと、LINE IDが自動記録されます。  
その後、管理者は以下を確認してください：

1. 管理者設定 → スタッフタブを開く
2. 該当スタッフの「LINE ID」欄に値が入っているか確認
3. 「登録名」が正しく設定されているか確認（未設定の場合は入力して変更）

> **LINE IDが自動連携される条件**: 登録名とLINE表示名が一致する場合は自動でリンクされます。異なる場合は手動で確認が必要です。

### スタッフ情報の編集

各スタッフ行を直接編集できます：

| フィールド | 説明 |
|-----------|------|
| 登録名 | アプリ内で表示される名前（管理者が設定） |
| LINE名 | LINEプロフィールから自動取得（参照用・変更不要） |
| メール | 連絡用メールアドレス |
| 有効 | チェックを外すとシフト一覧から非表示になる |
| LINE ID | LINE連携済みの場合に自動表示 |

### スタッフを無効化する

退職・長期休暇などで一時的にシフト対象外にする場合：
- 「有効」チェックボックスをオフにする（削除せずに残せる）

### スタッフを削除する

- 各スタッフ行の「削除」ボタンをタップ → 確認ダイアログで削除

## 5. シフト管理

### シフト希望の確認

**アクセス**: シフト管理 → 「希望一覧」タブ

1. 年月を選択
2. フィルターで絞り込み

| フィルター | 内容 |
|-----------|------|
| 全件 | すべての日付を表示 |
| 充足 | OK・応相談が2名以上 |
| 人手不足 | OK・応相談が1名以下 |
| 要注意 | 応相談スタッフがいる日 |

### 確定シフトの入力

**アクセス**: シフト管理 → 「確定入力」タブ

1. 年月を選択
2. 各日付行でスタッフと業務種別（業務A / 業務B）を選択
3. 「＋ 追加」で同日に複数スタッフを割り当て可能
4. 「確定シフトを保存」ボタンで一括保存

### 特定日の希望スタッフを確認する

「確定入力」タブ内の下部エリアで日付を選ぶと、その日の希望スタッフ一覧が表示されます。

### LINEグループへシフト通知を送信する

**アクセス**: シフト管理 → 「確定入力」タブ → 下部「LINEグループへシフト通知を送信」

1. 送信日を選択
2. 「今すぐ送信」ボタンをタップ

送信メッセージ例：
```
【シフト連絡】
2026-06-07
本日の担当は
田中さん（業務A）
鈴木さん（業務B）
です。よろしくお願いします。
```

> **前提**: LINE設定でChannel Access TokenとGroup IDが設定済みであること。

## 6. 基本設定

**アクセス**: 管理者設定 → 「基本設定」タブ

| 設定項目 | 説明 | 例 |
|---------|------|-----|
| 管理者メールアドレス | 通知メールの送信先 | admin@gmail.com |
| CCメールアドレス | 複数宛先（1行1アドレス） | cc1@gmail.com |
| 起床確認の期限時刻 | この時刻を過ぎると未確認アラートを送信 | 08:00 |
| LINE定型文の送信時刻 | GASトリガーの設定に合わせる | 07:00 |
| シフト対象曜日 | シフト提出・確定の対象曜日 | 月〜金 |

### パスワードの変更

1. 「新しいパスワード」欄に新しいパスワードを入力
2. 「確認用パスワード」欄にもう一度入力
3. 「設定を保存」ボタンをタップ

## 7. LINE設定

**アクセス**: 管理者設定 → 「LINE設定」タブ

### 必要な設定値

| 設定項目 | 取得場所 |
|---------|---------|
| Channel Access Token | LINE Developers → チャネル → Messaging API → Channel access token |
| Group ID | LINE BotをグループへINVITEし、Webhookログ等から確認 |

### 設定手順

1. 上記2つの値を取得
2. 管理者設定 → LINE設定タブに入力
3. 「保存」ボタンをタップ

## 8. 各種通知について

### 自動送信される通知一覧

| タイミング | 通知先 | 内容 |
|-----------|--------|------|
| スタッフが出勤ボタンを押したとき | 管理者メール | 出勤通知（時刻付き） |
| スタッフが退勤ボタンを押したとき | 管理者メール | 退勤通知（時刻付き） |
| トラブル第一報が送信されたとき | 管理者メール + CC | 緊急第一報 |
| トラブル第二報が送信されたとき | 管理者メール + CC | 緊急第二報（詳細付き） |
| 起床確認の期限を過ぎた未確認スタッフがいるとき | 管理者メール | 起床未確認アラート |
| 毎朝（GASトリガー設定時） | LINEグループ | 本日のシフト通知 |

> メール通知はGmailのMailAppを使用。無料Gmailは1日100通まで。

## 9. トラブルシューティング

### スタッフのLINEと登録名が連携されない

- 管理者設定 → スタッフタブで「LINE ID」欄が「未連携」になっていないか確認
- スタッフに「LINEからアプリを開いた」か確認する
- LINE表示名と登録名が一致していない場合は自動連携されないため、手動で登録名を設定する

### 管理者へのメールが届かない

- 管理者設定の「管理者メールアドレス」が正しく入力されているか確認
- 迷惑メールフォルダを確認
- Gmail1日100通の上限に達していないか確認

### LINEグループへの送信が失敗する

- Channel Access Tokenの有効期限を確認（LINE Developersで再発行）
- LINE BotがグループにINVITEされているか確認
- Group IDが正しいか確認

### 管理者パスワードを忘れた

Googleスプレッドシートの「管理者設定」シートを直接開き、`admin_password` の値を確認・変更してください。
